# 阶段 4：文档生成

前提：阶段 3 已完成，文档拆分方案已确认。

## 目标

按 source 范围分组，并行启动 Task 生成所有文档。

## 输入来源

| 来源 | 内容 | 用途 |
|------|------|------|
| 阶段 2 | 可文档化内容清单 | 确定 source 参考 |
| 阶段 3 | 文档拆分方案 | 确定文档职责、章节、对应源码范围 |
| 阶段 3 | index.md | 文档导航入口 |

---

## Task 分组策略

**核心原则**：参考相同 source 文件的文档归为同一组，由一个 Task 负责编写。

### 分组算法

```
1. 遍历阶段 3 的文档拆分方案
2. 提取每个文档的 source 参考集合
3. 按 source 集合相似度分组：
   - 完全相同 → 同一组
   - 高度重叠（>80%）→ 同一组
   - 无重叠 → 不同组
4. 每组包含 1~N 个文档
```

### 分组示例

```
组 A（参考 source/github/core/）：
  - concepts/core.md
  - guides/quickstart.md
  - api/main-api.md

组 B（参考 source/github/plugins/）：
  - plugins/overview.md
  - plugins/development.md

组 C（参考 source/npm/utils/）：
  - utils/helpers.md
```

---

## 并行控制

- **每批最多 3 个 Task 并行**
- 批次串行执行：前一批全部完成后再启动下一批
- 避免上下文爆炸和资源竞争

```
批次 1：Task A, Task B, Task C（并行）
     ↓ 等待全部完成
批次 2：Task D, Task E（并行）
     ↓ 等待全部完成
...
```

---

## 单个 Task 的 prompt 结构

每个 Task 通过 `Agent` 工具启动（`subagent_type: general-purpose`）。

### 1. 任务定义

```
你的任务：编写以下文档文件（共 N 个）

| 文件路径 | 职责 | 章节结构 |
|----------|------|----------|
| concepts/core.md | 解释核心概念 | 状态机 → 操作路径 → 失败地图 |
| guides/quickstart.md | 最小上手闭环 | 前置状态 → 完整示例 → 结果状态 → 常见失败 |
| api/main-api.md | API 参考 | 调用示例 → 参数 → 返回值 → 错误 → 状态影响 |
```

### 2. 参考源码

```
参考以下 source 文件：

source/github/core/
├── index.ts — 入口导出
├── engine.ts — 核心引擎
└── types.ts — 类型定义

约束：
- 只参考上述目录下的文件
- 禁止引用其他目录或外部文档
- 每个 source 引用必须精确到行号
```

### 3. 文档规则

```
【核心定位】
文档不是说明书，是：状态机 + 可执行操作路径 + 失败地图
文档读者是"另一个 AI agent" — 假设无任何额外上下文，文档必须自洽、封闭、可独立使用

【结构原则】
- 示例是主干，解释是附属
- 解释必须被示例证明，否则删除
- 示例与解释冲突时，以示例为准

【示例规则 — 主干】
每个示例必须是可执行的最小闭环：
- 初始状态（调用前系统处于什么状态）
- 完整调用（可直接复制执行的代码）
- 返回结果（实际输出，非伪代码）
- 状态变化（调用后系统变成了什么）

禁止：伪代码、不完整 JSON、未定义占位符、省略号

【状态机描述】
对于有状态的模块，必须明确：
- 可能的状态有哪些
- 状态转换条件
- 每个状态下可执行的操作
- 非法状态/转换的后果

【失败地图】
错误/约束与接口同级出现，必须明确：
- 触发条件（什么输入/状态会触发）
- 后果（抛什么错/进入什么状态）
- 是否可恢复（如何恢复）

【参数规则】
每项参数必须明确：必填性、类型、默认值、行为影响
每项必须附 source 标记

【溯源格式】
[陈述内容] `source: source/{类型}/{路径}:行号`

【禁止行为】
- 禁止基于"常见设计"补写参数行为
- 禁止假设默认值（必须从源码确认）
- 禁止推断错误类型
- 禁止无示例支撑的解释
- 禁止使用模糊词："通常""一般""建议""可能"
```

### 4. 输出要求

```
按顺序写入每个文档文件。

完成后报告：
| 文件 | 行数 | source 标记数 | 未确认项 |
|------|------|---------------|----------|
| concepts/core.md | 120 | 15 | 0 |
| guides/quickstart.md | 80 | 8 | 1 |
| api/main-api.md | 200 | 32 | 0 |

如有未确认项，说明已检查位置和无法确认的原因。
```

## 执行流程

1. **读取输入**：阶段 2 内容清单 + 阶段 3 拆分方案
2. **分组**：按 source 参考集合对文档分组
3. **分批调度**：每组一个 Task，每批最多 3 个并行
4. **执行**：启动批次 → 等待完成 → 下一批次
5. **汇总**：检查遗漏/冲突，更新 index.md 链接

---

- 所有文档文件已生成
- 每个事实性陈述已附 source 标记
- index.md 链接已指向实际文档

## 完成条件

全部文档生成完毕，用户确认后进入阶段 5。
