# 阶段 3：文档结构设计

前提：阶段 2 已完成，系统模型和内容清单已确认。

## 目标

设计以 index.md 为入口的文档导航体系。

## 3.1 index.md 设计

index.md 是根目录唯一的 md 文件，作为文档体系入口。

必须包含：
- 模块/概念的全景地图
- 到各子文档的链接 + 每个链接的"何时需要阅读"
- 不展开任何实现细节

**禁止**：嵌入代码示例、详细解释概念、冗余描述

示例结构：

```markdown
# [模块名] 文档地图

## 核心概念
- [概念A](concepts/a.md) - 当需要理解 X 时阅读
- [概念B](concepts/b.md) - 当遇到 Y 错误时查阅

## 快速开始
- [最小示例](guides/quickstart.md) - 第一次使用必读

## 深入主题
- [配置说明](guides/config.md) - 需要定制化时
```

## 3.2 单文档规模约束

- 硬性上限：1000 行，预警阈值：800 行
- 信息聚合度：同一主题相关信息集中比例 > 80%
- 拆分原则：按职责边界拆分（非简单按长度切分），新文档必须独立可使用

## 3.3 文档章节结构

每个文档应包含：
- 概述
- 示例（生产可用的最小闭环）
- 参数说明（每项附 source）
- 错误与约束（附 source）
- 关联文档链接

并说明每一章节对应的源码范围。

## 3.4 文档链接规则

- 使用**相对路径**
- 链接必须包含描述文字：`[描述文字](path/to/file.md)`
- 描述文字应说明"何时需要阅读"

## 本阶段输出

- index.md 完整内容
- 文档拆分方案（文件列表 + 每个文件的职责和章节大纲）

## 完成条件

文档结构和拆分方案用户确认后，进入阶段 4。
