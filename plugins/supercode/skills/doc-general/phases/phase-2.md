# 阶段 2：并行源码探索与系统建模

前提：阶段 1 已完成，所有源码已本地化。

## 目标

通过并行探索建立**可被文档化的系统模型**，完成证据溯源。

## 探索任务拆分

根据源码实际情况拆分为多个独立探索任务：
- 按模块/目录/职责边界拆分
- 每个部分启动一个 Explore Task（`subagent_type: Explore`）
- Task 之间无依赖，可并行执行

## 单个 Explore Task 输出格式

```markdown
## [探索部分] 报告

### 发现项 (N条)
- [内容] `source: source/.../file:行号`

### 未确认项 (N条)
> **未确认**: [内容]
> **已检查**: [已检查的文件]
> **原因**: [源码未声明 | 依赖包不可见 | 动态生成无法静态分析]
```

## 汇总阶段

所有 Explore Task 完成后：
1. 汇总各报告
2. 交叉验证冲突项
3. 整合为统一系统模型
4. 输出可文档化内容清单 + 不确定点列表

## 本阶段输出

- 系统模型概要
- 可文档化内容清单（每项附 source）
- 不确定点列表

## 完成条件

系统模型已建立，内容清单完整，用户确认后进入阶段 3。
