# 阶段 5：多 Task 并行验证

前提：阶段 4 已完成，所有文档已生成。

## 目标

启动多个独立 Task 并行验证文档正确性。

## Task 拆分原则

- 按文档章节/模块拆分
- 每个 Task 只检查一个独立部分
- Task 之间无依赖，可并行执行

## 单个 Task 检查内容

1. 读取该章节的所有 `source:` 标记
2. 逐一验证标记指向的 `source/` 文件:行号存在
3. 验证源码语义与文档描述一致
4. 验证文档内链接有效性（相对路径、目标文件存在）
5. 输出验证结果

## Task 输出格式

```markdown
## [章节名] 验证报告

### 已验证 (N条)
- [内容] <- `source: source/.../file:行号` 确认一致

### 不一致 (N条)
- [文档内容] <- `source: source/.../file:行号`
  - 源码实际：[真实行为]

### 无法验证 (N条)
- [内容] <- `source: source/.../file:行号`
  - 原因：[文件不存在 | 行号超出范围 | 无访问权限]
```

## 验证完成后

1. 汇总所有 Task 报告
2. 根据"不一致"和"无法验证"项修正文档
3. 重新验证修正部分（可迭代）
4. 全部通过后，任务完成

## 完成条件

所有验证通过，文档修正完毕，用户确认后任务结束。
