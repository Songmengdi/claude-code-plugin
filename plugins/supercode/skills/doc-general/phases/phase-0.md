# 阶段 0：需求探索

**禁止跳过本阶段。**

## 目标

通过持续提问，明确探索范围和文档目标。

## 执行方式

使用 AskUserQuestion 与用户对话，逐步明确：

- 需要探索什么？
- 涉及哪些 GitHub 仓库？npm 包？
- 是否需要网络搜索？搜索什么？
- 最终文档的目标形态？

## 执行原则

- 不遵循固定流程，根据用户回答动态追问
- 每次提问聚焦一个维度
- AskUserQuestion 优先使用多选项（multiSelect: true）
- 直到双方对"探索范围"和"文档形态"有清晰共识

## 本阶段输出

- 明确的探索范围
- 明确的文档目标

## 完成条件

双方对范围和目标达成共识，用户确认后进入阶段 1。
