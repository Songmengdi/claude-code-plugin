# 阶段 5：验收测试和提交

**前提：所有任务已完成。**

## 目标

运行完整的验收测试，通过后提交代码。

## 执行方式

### 5.1 运行验收测试

**执行顺序（重要）**：

**1. 任务测试**
检查计划中每个任务的"测试"部分，按顺序执行：
- 单元测试：执行计划中指定的命令
- 集成测试：执行计划中指定的命令
- 用户协助验证：提示用户按计划中的步骤验证

**2. 验收标准**
如果计划头部定义了验收标准，逐一执行：
- ✅ 标记为通过的验收标准
- ❌ 标记为失败的验收标准

**3. 项目测试套件**（可选）
```bash
npm test / cargo test / pytest / go test ./...
```

### 处理测试结果

**如果任何测试失败**：
- 停止执行
- 报告失败的测试（测试输出、错误信息、哪个任务/验收标准失败）
- **停止并等待用户修复指示**

**如果所有测试通过**：
- 继续到 5.2

### 5.2 更新文档状态并提交

**1. 更新 plan 文档的 frontmatter**：
- 将 `status` 从 `plan` 更新为 `executed`

**2. 提交所有代码和文档**（使用中文提交信息）：
```bash
git add .
git commit -m "feat(<功能名>): 完成功能实现

- 实现了 <具体功能1>
- 实现了 <具体功能2>
- 通过所有验收测试"
```
