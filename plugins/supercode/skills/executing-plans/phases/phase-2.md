# 阶段 2：执行批次

**前提：阶段 1 已完成，计划审查通过。**

## 目标

执行一批任务（默认前 3 个）。

## 执行方式

### 批次大小

**默认：前 3 个任务**

剩余任务少于 3 个时，执行所有剩余任务。

### 对每个任务

1. 标记为 in_progress
2. 严格遵循计划中的步骤
3. **按计划中提供的测试方式验证**：
   - 单元测试：执行计划中指定的命令
   - 集成测试：执行计划中指定的命令
   - 用户协助验证：提示用户进行验证
   - **不要随意测试，必须使用计划中指定的测试方式**
4. 标记为 completed

**重要：不要在每个任务后提交 git**，所有提交将在验收通过后统一进行。

- 已完成的任务

## 完成条件

批次中所有任务完成，**停止并等待用户回应**。
