# 阶段 1：加载和审查计划

## 目标

加载计划文件，进行批判性审查。

## 执行方式

### 1.1 读取计划文件

读取计划文件路径：
- 计划文件：用户提供的路径
- 设计文档：从 frontmatter 的 `related.design` 读取（可选）

### 1.2 批判性审查

检查以下内容：

| 审查项 | 检查内容 |
|--------|----------|
| 计划完整性 | 是否有遗漏或疑虑 |
| 验收标准 | 是否明确、可执行 |
| 测试方式 | 每个任务的测试方式是否明确 |
| 单元测试 | 是否有具体的测试命令 |
| 集成测试 | 是否有具体的测试命令 |
| 用户协助验证 | 是否有详细的验证步骤 |

### 1.3 处理审查结果

**如果有疑虑**：
- 在开始前与用户提出
- 等待用户回应

**如果没有疑虑**：
- 创建 TodoWrite 任务清单
- 继续下一阶段

## 任务清单

```markdown
[
  {"content": "读取计划文件", "status": "in_progress", "activeForm": "正在读取计划文件"},
  {"content": "批判性审查计划", "status": "pending", "activeForm": "正在审查计划"},
  {"content": "处理审查结果", "status": "pending", "activeForm": "正在处理审查结果"}
]
```

## 本阶段输出

- 计划审查结果
- 待执行的任务清单

## 完成条件

审查完成，**停止并等待用户回应**。
