# 阶段 0：加载功能上下文

## 目标

在新会话中快速了解现有功能的状态和历史变更。

## 何时执行

- 用户提到"改一下X功能"或"修复Y bug"
- 会话历史对此功能无记录
- 用户明确说明要改的功能或提供文档路径

## 何时跳过

- 用户已在当前会话中讨论过此功能
- 用户明确表示这是全新功能
- 功能需求已经明确

## 执行方式

使用 `Agent` 工具调用 feature-context-loader 子代理：

```yaml
subagent_type: "supercode:feature-context-loader"
prompt: |
  加载功能上下文：
  - 功能名称/标识：[用户提供的功能名或路径]
  - 目标：了解该功能的现状和历史变更
```

## 判断与确认

根据子代理返回的 Markdown Table 格式上下文摘要，使用 `AskUserQuestion` 确认：

- 上下文是否正确？
- 是否需要加载其他功能？
- 这是新功能还是现有功能的变更？

## 本阶段输出

- 明确的功能上下文
- 对"新功能"vs"现有功能变更"的共识

## 完成条件

上下文确认无误，用户同意进入下一阶段。
