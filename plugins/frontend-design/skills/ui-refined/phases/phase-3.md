# 阶段 3：验证

## 目标

请用户在 Pencil 中目视检查设计稿，发现问题则回到实现阶段修正。

## 步骤

### 3.1 请求用户目视检查（首选）

**这是最重要的验证方式。** 设计稿是协作产物，用户的视觉判断最快最准。

每完成一个页面后，直接请求用户在 Pencil 中查看，附上检查项清单：

- [ ] 无重复元素（同一区域出现相同内容）
- [ ] 无布局重叠（页面之间、组件之间）
- [ ] 对齐一致（同层级组件对齐方式统一）
- [ ] 文字无截断
- [ ] 间距一致（同类组件间距相同）
- [ ] 说明卡片与页面对应且内容完整
- [ ] 变体状态与说明卡片描述一致

根据用户反馈直接修正问题，回到实现阶段。

### 3.2 导出图片 + 自动化分析（仅用户要求时）

**仅在用户主动要求时才使用。** 不要自行导出图片分析。

**不要完全信任分析结果。** analysis-images 可能漏报，关键问题应以用户判断为准。

### 3.3 深度检查（可选）

当目视检查不够时，用脚本深入检查节点结构。

## 完成条件

所有页面通过用户目视检查，或问题已修正并重新验证通过。
