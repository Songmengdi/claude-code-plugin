# 阶段 2：实现

## 目标

逐页绘制精细化 UI，每页附带说明卡片。

## 绘制规则

### 分批绘制（强制）

batch_design 有原子性：一条失败，整批回滚。每个页面必须分批调用，不要试图一次完成。

**规则：**
- **每批最多 8-10 个操作**
- 按层级从外到内拆分：先骨架 → 再逐区域填充内容 → 最后细节调整
- 跨批必须用上批返回的**真实节点 ID** 作为父节点（绑定仅在同批内有效）
- 失败时立即 `get_editor_state` 检查残留节点，用 `D()` 清理

```
# 第1批：页面骨架（3-5个操作）
frame = I("", {"type": "frame", "width": 1920, "height": 1080, ...})
nav = I("frameId", {"type": "frame", ...})       # 用上批返回的真实ID
body = I("frameId", {"type": "frame", ...})
sidebar = I("frameId", {"type": "frame", ...})

# 第2批：导航栏内容（5-8个操作）
logo = I("navId", {"type": "text", ...})
search = I("navId", {"type": "frame", ...})

# 第3批：侧边栏内容
# ...（用 sidebar 的真实 ID）
```

### 图标绘制

必须使用 `icon_font` 类型 + Lucide 字体。独立图标可直接放在 flex 容器中，文本+图标组合需包裹在子 frame 中。

### 文本换行

Pencil 不支持 `flexWrap`，需手动将长文本拆分为多个 text 节点。

### 组件参考尺寸

绘制时参照 PC 端和移动端组件推荐尺寸表。

## 变体行绘制

主页面只画默认态。变体涵盖**组件级**（按钮 hover/disabled）和**区域级**（ToolCall 展开/折叠、对话区加载态、输入框附件态等）两类。

**通过 AskUserQuestion 与用户头脑风暴**，确认需要展示哪些组件/区域的哪些变体状态。

绘制完所有变体后，使用 `auto_layout.py` 自动排列间距。

## 页面说明卡片

每个页面右侧必须有说明卡片，记录草图和设计稿中无法表达的细节。

| 属性 | PC 说明 | Phone 说明 |
|------|---------|------------|
| 宽度 | 280px | 280px |
| 边框色 | `#4ecdc4`（青绿） | `#ff6b6b`（红色） |

### 说明卡片内容

只写设计稿无法表达的信息：

1. **布局行为** — 固定区域 vs 可滚动区域、吸顶/吸底
2. **交互说明** — 悬浮效果、点击行为、动画
3. **状态说明** — 简要列出关键组件的交互状态，详细变体见下方变体行
4. **响应式断点** — 如果同时有 PC 和 Phone 版本

## 节点查询

实现过程中频繁需要查询节点状态。默认直接读 .pen 文件，不经过 batch_get。

## 完成条件

所有页面绘制完成，每页附带说明卡片，**进入验证阶段**。
