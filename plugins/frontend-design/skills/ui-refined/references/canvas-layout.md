# 画布布局与坐标规则

## 固定尺寸

| 设备 | 尺寸 |
|------|------|
| PC 端 | 1920 × 1080 |
| 移动端 | 375 × 812 |

## 排列公式

横向：**页面 → 说明卡片**，纵向按组排列。每组间距 200px。

**风格卡片**（如果存在）放在 y=0。

### 仅 PC

| 元素 | x | y |
|------|---|---|
| 第 N 组 PC | 0 | (N+1) × 1280 |
| 第 N 组 PC 说明 | 2020 | (N+1) × 1280 |

### 仅 Phone

| 元素 | x | y |
|------|---|---|
| 第 N 组 Phone | 0 | (N+1) × 1012 |
| 第 N 组 Phone 说明 | 475 | (N+1) × 1012 |

### PC + Phone

| 元素 | x | y |
|------|---|---|
| 第 N 组 PC | 0 | (N+1) × 1280 |
| 第 N 组 PC 说明 | 2020 | (N+1) × 1280 |
| 第 N 组 Phone | 2400 | (N+1) × 1280 |
| 第 N 组 Phone 说明 | 2880 | (N+1) × 1280 |

> N 从 0 开始。(N+1) 是因为 y=0 预留给风格卡片。1280 = 1080 + 200，1012 = 812 + 200。

## 变体行定位

主页面只画**默认态**。变体涵盖两类：

1. **组件级变体** — 按钮（hover/disabled）、输入框（聚焦/错误/空态）等小组件的状态变化
2. **区域级变体** — 整个功能区域的状态变化，如列表的加载态、侧边栏的展开/折叠态、表单的错误态等

变体以**组件/区域变体**形式画在页面下方，称为**变体行**。

| 属性 | 规则 |
|------|------|
| y 位置 | 紧跟页面组最底部 + 100px 间距 |
| x 位置 | 从 0 开始，与所属页面对齐 |
| 变体尺寸 | **与主页面中该组件/区域的宽高完全一致**，不可随意缩放。区域级变体必须保持与主页面对应区域相同的宽度 |
| 命名 | 通过 `name` 属性标注状态，格式 `ComponentName/State`（如 `Button/Hover`），不需要额外的 label 节点 |
| 背景 | 变体必须包含所属区域的背景色（如侧边栏组件用 `$$--card`），确保状态差异可见 |
| 数量 | 一般 2-5 个，超过则分多行（行间距 100px） |

### 排列方式

所有变体统一**纵向排列**，使用 `auto_layout.py` 脚本自动计算间距。

```
┌──────────────────────┐  ┌──────────┐
│   PC 页面（默认态）    │  │ PC说明    │
│   (1920×1080)        │  │          │
└──────────────────────┘  └──────────┘
┌────────────────────────────┐
│ 按钮-hover                  │  ← 变体行
└────────────────────────────┘
┌────────────────────────────┐
│ 按钮-disabled               │
└────────────────────────────┘
┌────────────────────────────┐
│ 按钮-active                 │
└────────────────────────────┘
```

> 脚本用法见 `references/node-operations.md`「自动布局脚本」章节。

> 变体行不计入页面高度，但会影响下一组页面的 y 偏移。
> 计算下一组位置时，必须取 `max(页面底部, 变体行底部) + 200px`。

## 说明卡片

每个页面右侧必须有说明卡片，记录草图和设计稿中无法表达的细节。

| 属性 | PC 说明 | Phone 说明 |
|------|---------|------------|
| 宽度 | 280px | 280px |
| 边框色 | `#4ecdc4`（青绿） | `#ff6b6b`（红色） |

### 说明卡片内容

只写设计稿无法表达的信息：

1. **布局行为** — 固定区域 vs 可滚动区域、吸顶/吸底
2. **交互说明** — 悬浮效果、点击行为、动画
3. **状态说明** — 简要列出关键组件的交互状态，详细变体见变体行
4. **响应式断点** — 如果同时有 PC 和 Phone 版本

## 组件变体规范

同类组件的不同变体之间**只改必要属性**，其余继承基础样式：

| 变体类型 | 差异维度 | 示例（Button） |
|----------|----------|---------------|
| 类型变体 | `fill` + `stroke` | Default: `fill: $$--primary`; Ghost: 无 fill，只有 stroke |
| 尺寸变体 | `height` + `padding` | standard: height 40, padding [10,16]; Large: height 48, padding [12,24] |

通用组件标记为可复用，命名格式 `ComponentName/Variant`：

```
btn = I("", {"type": "frame", "name": "Button/Default", "reusable": true, ...})
btnGhost = I("", {"type": "frame", "name": "Button/Ghost", "reusable": true, ...})
```
