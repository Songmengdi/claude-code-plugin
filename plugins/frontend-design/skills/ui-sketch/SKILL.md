# UI 草图绘制

通过 excalidraw MCP 工具与用户协作，从需求沟通到草图产出。

## 核心原则

1. **探索优先** — 先了解工具能力，再动手
2. **用户驱动** — 需求和布局决策必须与用户确认，不要猜测
3. **管道过滤** — MCP 返回数据必须用 jq 过滤，禁止直接使用原始输出
4. **批量绘制** — 用 batch_create_elements 一次画完一个页面

## 阶段化工作流程

本技能分为 8 个阶段，每个阶段的具体指令存储在独立文件中。

**当前你只处于一个阶段。禁止猜测其他阶段的内容。**

| # | 名称 | 详情 |
|---|------|------|
| 1 | 了解工具 | `phases/phase-1.md` |
| 2 | 沟通需求 | `phases/phase-2.md` |
| 3 | 检查画布 | `phases/phase-3.md` |
| 4 | 计算布局与绘制页面 | `phases/phase-4.md` |
| 5 | 绘制说明卡片 | `phases/phase-5.md` |
| 6 | 绘制变体行 | `phases/phase-6.md` |
| 7 | 展示初稿与迭代 | `phases/phase-7.md` |
| 8 | 保存草图 | `phases/phase-8.md` |

### 执行规则（强制）

1. **每个阶段的第一步必须是 `Read` 对应的 `phases/phase-N.md`**，未读取前禁止执行任何阶段操作
2. 阶段完成后 **停止并等待用户回应后再进入下一阶段**
3. 阶段 6 为可选阶段，无变体需求时跳过
4. 迭代修改时（阶段 7），只需重新读取受影响的阶段文件，不需要从头开始
