# 阶段 6：绘制变体行

## 目标

展示组件的多种状态（hover、active、disabled、空态、加载态、错误态等）。

## 流程

1. 分析当前页面的关键交互组件
2. **通过 AskUserQuestion 与用户头脑风暴**，确认需要展示哪些组件的哪些变体状态
3. 按规则绘制变体行

## 变体区规则

主页面只画**默认态**。其他状态以**组件变体**形式画在页面下方，称为**变体行**。

变体是**组件级别**的（如按钮的 hover/disabled、输入框的聚焦/错误/空态等），不是整个页面的缩略图。

### 布局示意

```
┌──────────────────────┐  ┌──────────┐
│   PC 页面（默认态）    │  │ PC说明    │
│   (1920×1080)        │  │          │
└──────────────────────┘  └──────────┘
┌────────────┐ ┌────────────┐ ┌────────────┐
│ 按钮-hover │ │ 按钮-disabled│ │ 按钮-active │  ← 变体行（组件级）
└────────────┘ └────────────┘ └────────────┘
```

### 排列方向

| 组件宽度 | 排列方式 |
|----------|----------|
| 窄组件（如按钮、输入框） | **横向排列**，组件之间 40px |
| 宽组件（如卡片、表单区块） | **纵向排列**，组件之间 60px |

### 定位规则

| 属性 | 规则 |
|------|------|
| y 位置 | 紧跟页面组最底部 + 100px 间距 |
| x 位置 | 从 0 开始，与所属页面对齐 |
| 变体尺寸 | **与主页面中该组件的宽高一致**，不可随意缩放 |
| 标签 | 每个变体上方标注状态名称 |
| 数量 | 一般 2-5 个，超过则分多行（行间距 100px） |

### 对下一页面的影响

> 变体行不计入页面高度，但会影响下一组页面的 y 偏移。
> 计算下一组位置时，必须取 `max(页面底部, 变体行底部) + 200px`。

## 完成条件

变体行绘制完成，进入阶段 7。
