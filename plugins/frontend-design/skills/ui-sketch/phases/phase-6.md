# 阶段 6：绘制变体行（可选）

## 目标

展示组件的多种状态（hover、active、disabled、空态、加载态、错误态等）。

## 何时执行

- 阶段 2 中用户明确要求展示组件多状态
- 组件有重要的交互状态变化需要可视化

## 何时跳过

- 简单页面，组件状态单一
- 用户未提出变体需求

**读取本文件后，你才知道变体区的具体规则。**

## 变体区规则

主页面只画**默认态**。其他状态以缩略图形式画在页面下方，称为**变体行**。

### 布局示意

```
┌──────────────────────┐  ┌──────────┐
│   PC 页面（默认态）    │  │ PC说明    │
│   (1920×1080)        │  │          │
└──────────────────────┘  └──────────┘
┌────────┐ ┌────────┐ ┌────────┐
│ 变体A   │ │ 变体B   │ │ 变体C   │  ← 变体行
└────────┘ └────────┘ └────────┘
```

### 定位规则

| 属性 | 规则 |
|------|------|
| y 位置 | 紧跟页面组最底部 + 100px 间距 |
| x 位置 | 从 0 开始，与所属页面对齐 |
| 变体宽度 | 按 1920px（PC）或 375px（Phone）等分，间距 40px |
| 标签 | 每个变体上方标注状态名称 |
| 数量 | 一般 2-5 个，超过则分多行（行间距 100px） |

### 对下一页面的影响

> 变体行不计入页面高度，但会影响下一组页面的 y 偏移。
> 计算下一组位置时，必须取 `max(页面底部, 变体行底部) + 200px`。

## 完成条件

变体行绘制完成，进入阶段 7。
