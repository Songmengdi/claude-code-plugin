# 阶段 4：计算布局与绘制页面

## 目标

按坐标公式绘制页面（PC / Phone），一次完成一个页面的所有元素。

## 前置信息

**读取本文件后，你才知道以下信息。**

### 固定尺寸

| 设备 | 尺寸 |
|------|------|
| PC 端 | 1920 × 1080 |
| 移动端 | 375 × 812 |

### 页面排列规则

横向顺序：**页面 → 说明卡片**，多个页面纵向排列。

同一页面若同时有 PC 和 Phone 版本，横向依次排列，各自带独立说明卡片。

**仅 PC**：
```
┌──────────────────────┐  ┌──────────┐
│   PC 页面（默认态）    │  │ PC说明    │
│   (1920×1080)        │  │          │
└──────────────────────┘  └──────────┘
```

**仅 Phone**：
```
┌──────────┐  ┌──────────┐
│ Phone    │  │ Phone说明 │
│(375×812) │  │          │
└──────────┘  └──────────┘
```

**PC + Phone**：
```
┌──────────────────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐
│   PC 页面（默认态）    │  │ PC说明    │  │ Phone    │  │ Phone说明 │
│   (1920×1080)        │  │          │  │(375×812) │  │          │
└──────────────────────┘  └──────────┘  └──────────┘  └──────────┘
```

### 坐标计算公式

元素间距统一 **100px**，说明卡片宽度 **280px**。

**仅 PC 时**：

| 元素 | x | y |
|------|---|---|
| 第 N 组 PC | 0 | N × 1280 |
| 第 N 组 PC 说明 | 2020 | N × 1280 |

**仅 Phone 时**：

| 元素 | x | y |
|------|---|---|
| 第 N 组 Phone | 0 | N × 1012 |
| 第 N 组 Phone 说明 | 475 | N × 1012 |

**PC + Phone 时**：

| 元素 | x | y |
|------|---|---|
| 第 N 组 PC | 0 | N × 1280 |
| 第 N 组 PC 说明 | 2020 | N × 1280 |
| 第 N 组 Phone | 2400 | N × 1280 |
| 第 N 组 Phone 说明 | 2880 | N × 1280 |

> N 从 0 开始计数。1280 = 1080 + 200(组间距)，1012 = 812 + 200(组间距)。
> x 计算：PC(0) + 100 + 1920 = 2020，PC说明(2020) + 100 + 280 = 2400，Phone(2400) + 100 + 375 = 2880。

**有变体行时**，组间距需要加上变体行高度：

> **变体累计偏移** = 前面所有组的变体行额外高度之和。
> 示例：第 0 组 PC 有 1 行变体（高 200px），则第 1 组 PC 的 y = 1280 + 200 = 1480。

### 组件尺寸参考

**PC 端**：

| 组件 | 推荐高度 | 备注 |
|------|----------|------|
| 顶部导航栏 | 56-64px | 固定 |
| 侧边栏宽度 | 240-320px | 固定 |
| 文章卡片 | 120-180px | 含标题+摘要+日期 |
| 底部栏 | 200-300px | 页脚信息 |

**移动端**：

| 组件 | 推荐尺寸 | 备注 |
|------|----------|------|
| 顶部导航 | 44-56px | 含汉堡菜单 |
| 卡片内边距 | 16-20px | 左右留白 |
| 触控区域 | ≥ 44px | 按钮和链接 |

### 颜色规范

草图只表达框架布局，不需要精细配色。使用简单的灰色系区分结构层次：

| 用途 | 颜色 | 说明 |
|------|------|------|
| 页面外框 | `#333333` | 实线，表示页面边界 |
| 区域边框 | `#dddddd` | 细线，区分不同区域 |
| 文字 | `#333333` | 统一使用 |
| 页面背景 | `#ffffff` | |
| 内容区背景 | `#fafafa` | 与侧边栏区分 |

### 交互元素绘制

| 交互类型 | 绘制方式 | 说明 |
|----------|----------|------|
| 按钮点击 | 矩形 + 文字标签 | 如"提交"、"取消" |
| 悬浮菜单/下拉 | 虚线矩形 | 表示弹出层，标注触发条件 |
| 弹窗/模态框 | 半透明矩形覆盖 + 居中矩形 | 标注"点击XX弹出" |
| 侧边栏抽屉 | 箭头 + 滑动方向 | 标注"从左/右滑出" |
| 标签切换 | 一组矩形并排 + 底部高亮线 | 标注"Tab切换" |
| 手势操作 | 箭头指示方向 | 如"左滑删除"、"下拉刷新" |
| 展开收起 | 上下箭头 | 标注"点击展开/收起" |

> 交互元素的标注要简洁，详细行为写在说明卡片中。

## 绘制技巧

1. **单个页面先框架后细节**：先用 `batch_create_elements` 画容器框（外框、区域分隔），再用一次调用画内部元素
2. **文字用独立的 text 元素**叠加在矩形上方，不要写在矩形内部
3. **不同区域用背景色区分**（白色 vs 浅灰）即可，不需要精细配色
