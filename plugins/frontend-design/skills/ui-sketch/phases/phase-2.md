# 阶段 2：沟通需求

**最重要的一步。**

## 目标

明确要画什么，避免猜测。

## 原则

**不要猜测用户想要什么，去问。** 给用户 2-3 个方案选择，附上你的推荐和理由。

## 需要明确的要素

**页面信息**：
- 要画什么类型的页面？（博客、电商、管理后台、落地页…）
- 核心功能模块有哪些？
- 设计风格偏好？（极简、商务、活泼…）

**设备范围**：
- 需要 PC 端？移动端？还是都要？
- 有没有特殊的尺寸要求？

**布局决策**（常见需要确认的点）：
- 导航栏风格：横向导航 vs 侧边栏 vs 汉堡菜单？
- 侧边栏行为：固定显示 vs 可折叠 vs 放底部？
- 内容区域：固定高度 vs 可滚动？
- 有没有参考网站或竞品？

## 沟通示例

```
Q: 你希望侧边栏是固定在右侧，还是可以折叠？

Q: 移动端的分类和标签，你倾向于哪种方式？
   - 方案A：放页面底部，滚动到底部可见
   - 方案B：固定底栏，始终可见
   - 方案C：放入汉堡菜单中展开

Q: 导航栏需要固定在顶部吗？还是随页面滚动？
```

## 本阶段输出

- 页面类型和功能模块
- 设备范围（PC / Phone / 两者）
- 关键布局决策
- 组件是否需要展示多状态（变体）

## 完成条件

需求确认完毕，**停止并等待用户确认后再进入阶段 3**。
